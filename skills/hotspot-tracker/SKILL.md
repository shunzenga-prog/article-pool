# 热点追踪器

追踪 AI 领域热点，生成可创作的选题建议。

## 触发场景
- "今天有什么热点" / "追热点" / "找选题"
- 定时任务触发

## 快速使用
```bash
bash skills/hotspot-tracker/scripts/check-hotspots.sh
```

## 热点来源

### 📰 新闻来源
| 优先级 | 来源 |
|--------|------|
| P0 | 机器之心、量子位 |
| P1 | 36 氪、微博热搜、知乎热榜 |
| P2 | Hacker News、国际媒体 |

**详细清单**：见 `references/ai-news-platforms.md`

### 📚 学术来源
| 优先级 | 来源 |

**详细清单**：见 `references/academic-sources.md`

```markdown
# 热点追踪报告 - YYYY-MM-DD

## 📰 新闻热点 TOP 3
| 序号 | 热点 | 来源 | AI 相关性 |
|------|------|------|----------|
| 1 | xxx | 搜索 | ⭐⭐⭐ |

## 📚 学术论文 TOP 3
| 序号 | 论文 | 作者/机构 | 核心贡献 |
|------|------|----------|----------|
| 1 | xxx | MIT | 首次实现... |

## 可创作选题
1. **新闻选题** - 标题建议 | 角度：xxx
2. **学术选题** - 论文解读 | 角度：通俗解释 xxx
```

## 学术内容创作规范

### 选题公式
```
学术选题 = 论文贡献 + 通俗解释 + 行业影响 + 实操建议
```

### 学术诚信
- ✅ 准确引用：论文标题、作者、机构、arXiv ID
- ✅ 标注来源："论文解读"而非"新闻"
- ❌ 不曲解论文结论、不夸大影响

### 创作角度
1. **论文解读** - 通俗解释核心贡献
2. **技术对比** - 与已有方法对比
3. **实操指南** - 如何复现/使用
4. **行业影响** - 对行业的影响

## 注意事项（⚠️ 核心原则）

**搜索热点时，必须验证来源和时间！**
- 新闻：验证发布时间，>48h 转长青角度
- 论文：验证 arXiv 发布日期、作者机构
- 已辟谣/撤稿 → 不使用

**时效分类：**
| 发布时间 | 标注 | 是否可用 |
|----------|------|----------|
| < 24h | 🔥实时热点 | ✅ 优先使用 |
| 24-48h | 📈近期热点 | ✅ 可用 |
| > 48h | 💡长青话题 | ⚠️ 转分析角度 |
| 无法验证 | ❌ | 🚫 不使用 |

## ⚠️ 早报制作流程

### Step 1: 搜索热点
```bash
curl -s "https://search.brave.com/search?q=AI+news+$(date +%Y-%m-%d)"
```

### Step 2: 验证每条消息 ⚠️ 必须做
- [ ] 找到原文链接
- [ ] 确认发布时间（精确到天）
- [ ] 确认来源可靠（主流媒体/官方博客）
- [ ] 检查是否已辟谣/撤稿

### Step 3: 质量检查
- [ ] 每条消息都有**真实来源链接**
- [ ] 每条消息都有**精确发布时间**
- [ ] 没有无法验证的消息
- [ ] 没有超过 48h 当"实时热点"的消息

### ⚠️ 验证是幕后工作，不呈现给读者
**不要在早报内容中强调：**
- ❌ "每条新闻已验证真实性" / "小咪已帮你验证过" / "来源均为可靠媒体"

---

### Step 4: 润色（发布公众号前必须）

**派润色龙虾润色：**
```
sessions_spawn(label="polish-lobster", task="润色早报，准备发布到公众号...")
```

**润色要求（学习 CSDN 极客头条）：**
1. ❌ 不用 emoji（只保留署名处 1 个）
2. ❌ 不标注时间/来源（验证是幕后工作）
3. ❌ 不用表格（自然叙述）
4. ✅ 分类：国内要闻/国际要闻/程序员专区
5. ✅ 内容长度根据实际需要（简单新闻 2-3 句，重要新闻 4-5 句）
6. ✅ **输出 HTML 格式**（可直接粘贴公众号）

**输出**：`reports/ai-daily-brief-YYYY-MM-DD.html`

**HTML 模板**：见 `templates/daily-brief-template.html`

### 样式规范
