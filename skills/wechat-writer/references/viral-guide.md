## 一、标题技巧（决定 80% 打开率）

### 6 种爆款标题公式

| 类型 | 公式 | 案例 |
|------|------|------|
| **数字冲击** | 数字 + 核心信息 + 情绪词 | 《3 小时破 10 万，这个 AI 工具杀疯了》 |
| **对比反差** | 旧认知 vs 新事实 | 《你以为 AI 只会画图？它已经开始写代码了》 |
| **悬念提问** | 问题 + 暗示答案 | 《为什么大厂都在抢这个方向？答案在这里》 |
| **权威背书** | 大厂/名人 + 动作 + 影响 | 《OpenAI 深夜发文，AGI 要来了？》 |
| **紧迫时效** | 刚刚/突发 + 事件 + 解读 | 《刚刚，DeepSeek 发布新模型》 |
| **利益相关** | 你/我们 + 受益/受损 | 《这波 AI 红利，普通人怎么抓住？》 |

### 标题检查清单

- [ ] 是否包含数字或情绪词？
- [ ] 是否制造悬念或反差？
- [ ] 长度是否在 20-28 字？
- [ ] 是否与内容匹配？

## 二、开头写法（决定 50% 读完率）

### 黄金 3 秒法则

读者决定是否继续阅读只有 3 秒。

### 4 种开头方法

**1. 场景切入法**
```
凌晨 3 点，北京中关村的一间办公室里，
30 岁的程序员李明盯着屏幕上的代码，
手心全是汗。
```

**2. 数据冲击法**
```
10 万、50 万、100 万……

短短 72 小时，这个 AI 工具的注册用户
像坐火箭一样飙升。
```

**3. 金句开场法**
```
"AI 不会取代你，但会用 AI 的人会。"

这句话在 2026 年，比任何时候都更真实。
```

**4. 反差对比法**
```
三年前，他还在为找工作发愁；
三年后，他的公司估值 10 亿。
```

## 三、正文结构

### 4 种爆款结构

**A. SCQA 结构**（深度分析）
```
S（情境）：背景
C（冲突）：问题
Q（问题）：核心问题
A（答案）：解决方案
```

**B. 金字塔结构**（知识科普）
```
结论先行
├── 论点 1 + 论据
├── 论点 2 + 论据
└── 论点 3 + 论据
总结升华
```

**C. 故事线结构**（人物/事件）
```
起因 → 发展 → 高潮 → 结局 → 启示
```

**D. 清单体结构**（工具/方法）
```
引言
├── 方法 1 + 案例
├── 方法 2 + 案例
└── 总结 + 行动建议
```

## 五、写作风格

### 如何避免 AI 味道

| ❌ AI 味 | ✅ 人味 |
|---------|--------|
| 该工具具有以下特点 | 我用了一周，发现这工具有几个亮点 |
| DeepSeek 表现优异 | 说实话，当我看到它 30 秒生成完整项目时，我是真的被惊到了 |
| 此方法效率较高 | 这方法真的快，亲测有效 |
| 该功能受到欢迎 | 这个功能上线后，评论区炸了 |

### 人味写作技巧

1. **用第一人称** - "我用过"、"我发现"
2. **加入个人体验** - 真实故事、具体场景
3. **口语化表达** - 说人话，不说术语
4. **适当暴露情绪** - 惊讶、兴奋、无奈
5. **讲人话** - 用比喻代替术语

## 六、金句公式

| 公式 | 案例 |
|------|------|
| **对比式** | "AI 不会取代你，但会用 AI 的人会" |
| **递进式** | "选择比努力重要，但方向比选择更重要" |
| **反转式** | "你以为在学 AI，其实 AI 在学你" |
| **数字式** | "3 年积累，1 次爆发，这就是复利" |
| **比喻式** | "AI 是新时代的电力，谁掌握谁领先" |

### 金句打磨技巧

1. 删减冗余 - 去掉所有不必要的词
2. 制造节奏 - 长短句结合
3. 制造反差 - 前后对比
4. 具体化 - 用具体数字、场景
5. 多改几版 - 好金句是改出来的

---

## 七、爆款公式

```
爆款 = 好标题（40%）+ 好内容（40%）+ 好排版（20%）

好标题 = 数字/情绪 + 悬念/反差 + 利益相关
好内容 = 价值 + 故事 + 金句
好排版 = 清晰结构 + 舒适阅读 + 视觉友好
```
